## Code of Conduct

Be respectful, inclusive, and constructive. We are building something together.

---

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch from `main`
4. Make your changes
5. Open a pull request targeting `main`

---

## Branch Naming

Use the following format:

```
feat/short-description       # New feature
fix/short-description        # Bug fix
docs/short-description       # Documentation
chore/short-description      # Maintenance
refactor/short-description   # Code refactor
```

Examples:
- `feat/course-detail-page`
- `fix/enrollment-bug`
- `docs/update-readme`

---

## Commit Messages

Follow the [Conventional Commits](https://www.conventionalcommits.org/) standard:

```
feat: add course detail page
fix: resolve enrollment payment bug
docs: update API endpoint table
chore: upgrade dependencies
refactor: simplify auth middleware
```

---

## Pull Request Process

1. Ensure your branch is up to date with `main` before opening a PR
2. Fill out the PR template completely
3. Link the related issue in your PR description (e.g. `Closes #42`)
4. Make sure all checks pass (lint, build, tests)
5. Request a review from a maintainer
6. Do not merge your own PR — wait for maintainer approval

---

## Issue Guidelines

- Search existing issues before opening a new one
- Use the provided issue templates
- Be specific — include steps to reproduce for bugs
- For feature requests, explain the problem you are solving

---

## Questions?

Open a discussion or reach out to the maintainers via GitHub Issues.
